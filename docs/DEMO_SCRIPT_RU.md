# 🎬 SentinelCortex: Сценарий демо-презентации (MoD/B2B)

Этот документ содержит пошаговый план для записи видео-презентации системы безопасности SentinelCortex на русском языке.

## Шаг 1: Строгая компиляция (Стандарт безопасности)
**Команда:**
```bash
make sim-strict
```
**Что происходит:**
- Мы используем флаги `-Wall -Wextra -Wpedantic -Werror`.
- **Зачем:** Это гарантирует, что в коде **НЕТ** неопределенного поведения, неиспользуемых переменных или нестандартных конструкций C.
- **Результат:** Бинарный файл "военного уровня" — доказательство того, что код чист, надежен и готов к сертификации.

---

## Шаг 2: Автономная подпись (Суверенитет ключей)
**Команды:**
```bash
# Генерация мастер-ключей
./bin/rcf-signer keygen

# Подпись модуля A-Code
./bin/rcf-signer sign src/main.c logic.rcf
```
**Что происходит:**
- **KeyGen:** SentinelCortex генерирует пару ключей **Dilithium-2 (ML-DSA)** локально. Секретный ключ (Secret Key) никогда не покидает периметр клиента.
- **Подпись:** Утилита `rcf-signer` создает криптографический контейнер (`logic.rcf`), содержащий байт-код и постквантовую подпись.

---

## Шаг 3: Работа Sentinel VM (Защита в реальном времени)
**Команда:**
```bash
./bin/sentinel_sim
```
**Разбор логов:**
- `[RCF] Verifying module envelope...`: ВМ проверяет подпись модуля по зашитому **Мастер-ключу (MPK)**.
- `[RCF] Integrity VALID`: Подтверждение того, что модуль не был подменен или модифицирован.
- `[PQC] Initializing REAL Dilithium-2...`: ВМ выполняет реальные вычисления на решетках (NTT + Keccak) для постквантовой защиты.
- `[KEM] Initiating Kyber-512...`: Установление защищенной сессии с использованием квантово-устойчивого инкапсулирования ключей.
- `[REFLEX] USB_ENABLE pin pulled LOW`: Демонстрация "Рефлекторной безопасности" — ядро автономно отключает физические интерфейсы.
- `[HAL] FLASH: Erasing Sector 11...`: **Key Shredding** в действии. При обнаружении физического взлома система мгновенно уничтожает секретные ключи в энергонезависимой памяти, превращая устройство в "кирпич" для хакера.

## Шаг 4: Криптографический аудит RCF-CLI
**Команда:**
```bash
export RCF_LICENSE_KEY=RCF-AUDIT-ADMIN-KEY-GLOBAL
rcf-cli audit .
```
**Что происходит:**
- `rcf-cli` сканирует проект на наличие маркеров "защищенных активов".
- Проверяется соответствие исходного кода, Makefile и бинарных файлов эталонным хешам.
- **Результат:** `✅ RCF-Audit Complete`. Это финальное подтверждение безопасности перед развертыванием.
